<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Vite dev server (`front-end`, port 3000, HTTPS with repo certs) | Local static + HMR; proxies `/api`, `/ws`, `/grpc`, Connect paths to configured backend |
| Vite dev proxy → Go / gRPC backends | Only predefined path prefixes are forwarded; backend host from `VITE_API_ORIGIN` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | S | `server.proxy` target | mitigate | Keep explicit path keys in `front-end/vite.config.ts`; document `VITE_API_ORIGIN` in plan context; never derive proxy target from user input |
| T-01-02 | T | Local HTTPS certs (`.secrets/localhost+2*.pem`) | mitigate | Certs are local-only; `secure: false` toward dev backend is **dev-only** — do not copy to production Traefik/Nginx TLS config |
| T-01-03 | I | Static file serving via Vite | accept | No secrets in MPA HTML; same public assets as production Nginx |
| T-01-04 | E | WebRTC stubs in Vitest | accept | Test-only; no effect on production bundles |
| T-01-05 | D | Malformed request paths to pretty-route middleware | mitigate | Rewrite only exact pathnames `/react` and `/lit`; forward to fixed internal paths `react.html` / `index.html` |

**High-severity block:** If a task would add a proxy rule that forwards **arbitrary** URLs or user-controlled hosts, stop and redesign (ASVS L1 — no open proxy).
</threat_model>
