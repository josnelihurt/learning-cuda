<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Traefik → Nginx | TLS termination at edge; static files from `web-frontend`; only configured path prefixes reverse-proxied to Go |
| Nginx → go-app | Backend internal network; headers set as today (`X-Forwarded-*`) |
| Developer machine → Vite :3000 | Local TLS with repo-managed dev certs; proxies to configured backend |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | S | Nginx → `go-app` proxy | mitigate | Keep fixed `location` prefixes; no wildcards that turn Nginx into an open forwarder |
| T-02-02 | T | TLS dev certs | mitigate | Document that `.secrets/localhost+2*.pem` are **local only**; production TLS remains Traefik-managed |
| T-02-03 | I | Static asset tampering in image | accept | Build pipeline owns `dist/`; runtime image is read-only root except logs — integrity is supply-chain/build job |
| T-02-04 | D | Path confusion between `/lit` and `/react` | mitigate | Use **exact** `location =` matches; only rewrite to `index.html` / `react.html` |
| T-02-05 | E | Playwright preview server | accept | Local test-only; binds `127.0.0.1`; not exposed in production |

**High-severity block:** If Nginx changes introduce **recursive** `proxy_pass` loops, `alias`/`root` traversal outside `/usr/share/nginx/html`, or user-controlled redirect targets, stop and revert (ASVS L1).
</threat_model>
