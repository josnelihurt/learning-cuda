<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → same-origin Connect (`window.location.origin`, `/grpc` path) | Unary RPCs; cookies and CSRF posture unchanged from Lit |
| React hooks → `toast-container` DOM bridge | Untrusted strings are display text only (Lit template binding), not HTML injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | S | `createGrpcConnectTransport` / `baseUrl` | mitigate | **Hardcode** `baseUrl: window.location.origin` in factory only; **never** accept user-controlled origin strings in provider or hooks (ASVS L1) |
| T-02-02 | I | Connect unary responses | accept | Protobuf-encoded; integrity assumed via TLS same-origin deployment |
| T-02-03 | X | `useToast` title/message | mitigate | Pass through to existing Lit `ToastContainer` text nodes only; **do not** introduce `innerHTML` or HTML-sanitizer bypass in React bridge |
| T-02-04 | D | `useAsyncGRPC` error mapping | accept | Surface `ConnectError.message` / code for UI; no credential logging |

**High-severity block:** If any task would construct Connect transport or fetch `baseUrl` from **user input**, query params, or **remote config** (open redirect / open proxy), **stop and redesign** — ASVS L1 forbids ad-hoc cross-origin gRPC client configuration.
</threat_model>
