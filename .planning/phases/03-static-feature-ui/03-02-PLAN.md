  <name>Task 1: Create useImageProcessing hook</name>
  <files>front-end/src/react/hooks/useImageProcessing.ts</files>
  <action>
Create a custom hook `useImageProcessing` that handles image processing logic:

1. State management:
   - `processing: boolean` - processing in progress
   - `progress: number` - 0-100 progress percentage (simulated for now)
   - `processedImageUrl: string | null` - URL of processed image
   - `error: string | null` - error message

2. `processImage(imagePath: string, activeFilters: ActiveFilterState[])` function:
   - Validate inputs (imagePath not empty, activeFilters not empty)
   - Set `processing: true`, `progress: 0`, reset `processedImageUrl`
   - Start progress simulation (increment every 100ms until 90%)
   - Call `imageProcessorClient.processImage()` with:
     - `imagePath`: string from uploaded image
     - `filters`: map `ActiveFilterState[]` to `ActiveFilter[]` (same structure)
   - On success:
     - Set `processedImageUrl` from `response.processedImagePath`
     - Set `progress: 100`
     - Show toast success with `response.message`
   - On error:
     - Set error message
     - Show toast error with error details
   - Reset processing state after 300ms delay

3. Validation:
   - Show toast error if no image selected
   - Show toast error if no filters selected

4. `reset()` function:
   - Clear `processedImageUrl`, `error`, `progress`

5. Return: `{ processing, progress, processedImageUrl, error, processImage, reset }`

Follow the pattern from `front-end/src/react/hooks/useFilters.ts` for error handling and state management.
Import from `@/gen/image_processor_service_pb`: `ProcessImageRequest`, `ProcessImageResponse`.
Use `useServiceContext` from `@/react/context/service-context` to get `imageProcessorClient`.
Use `useToast` from `@/react/hooks/useToast` for notifications.
