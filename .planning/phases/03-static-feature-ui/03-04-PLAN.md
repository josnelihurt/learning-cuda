  <action>
Create a custom hook `useConfig` that handles fetching and updating system configuration:

1. State management:
   - `config: GetStreamConfigResponse | null` - current configuration
   - `loading: boolean` - loading state
   - `saving: boolean` - saving state
   - `error: string | null` - error message

2. `fetchConfig()` function:
   - Call `configService.getStreamConfig()` from existing service
   - On success: set `config` from response
   - On error: set error message, show toast error

3. `updateConfig(newConfig: GetStreamConfigResponse)` function:
   - Set `saving: true`
   - Call RPC to update configuration (check if `updateStreamConfig` exists)
   - On success:
     - Set `config` to new value
     - Show toast success
   - On error:
     - Set error message
     - Show toast error
   - Set `saving: false`

4. Return: `{ config, loading, saving, error, fetchConfig, updateConfig }`

Follow the pattern from `front-end/src/react/hooks/useFilters.ts` for error handling and state management.
Import types from `@/gen/config_service_pb`: `GetStreamConfigRequest`, `GetStreamConfigResponse`, `StreamEndpoint`.
Use existing config service from `@/application/services/config-service` or create gRPC client directly.
Use `useServiceContext` from `@/react/context/service-context` to get `configClient` (if exists).
Use `useToast` from `@/react/hooks/useToast` for notifications.

**Note:** If update RPC doesn't exist, implement `updateConfig` as a no-op or read-only, and add a TODO comment to implement when backend supports it.
