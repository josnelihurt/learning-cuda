    Create ReactFrameTransportService class at front-end/src/react/infrastructure/transport/ReactFrameTransportService.ts:

    **Class Structure (200+ lines):**
    - Constructor: Takes optional WebSocket URL, initializes connection state
    - Private properties:
      - ws: WebSocket | null
      - connectionState: 'connecting' | 'connected' | 'disconnected' | 'failed'
      - frameCallback: ((frameData: string) => void) | null
      - errorCallback: ((error: Error) => void) | null
    - Methods (all public):
      - initialize(): Promise<void> - Opens WebSocket connection to /ws/frame-transport
      - sendFrame(base64Data: string, filters: ActiveFilterState[]): void - Sends frame with serialized filters
      - setFrameCallback(callback: (frameData: string) => void): void - Registers callback for received frames
      - setErrorCallback(callback: (error: Error) => void): void - Registers callback for errors
      - getConnectionStatus(): string - Returns current connection state
      - close(): void - Closes WebSocket and clears callbacks
    - Private methods:
      - handleOpen(): void - Sets connection state to 'connected'
      - handleError(error: Event): void - Sets state to 'failed', calls error callback
      - handleClose(): void - Sets state to 'disconnected'
      - handleMessage(event: MessageEvent): void - Parses WebSocketFrameResponse, deserializes frame, calls callback

    **Frame Serialization (referencing Lit implementation):**
    - Use ProcessImageRequest protobuf (import from gen/frame_pb or equivalent)
    - Serialize ActiveFilterState[] to protobuf FilterParameter[]
    - Include sessionId, filter parameters in frame message
    - Base64 encode frame data before sending

    **Frame Deserialization:**
    - Parse WebSocketFrameResponse protobuf (import from gen/frame_pb or equivalent)
    - Extract processed_frame field (base64-encoded string)
    - Pass base64 data to frame callback (VideoCanvas will decode)

    **WebSocket Endpoint:**
    - Use wsUrl = '${protocol}//${window.location.host}/ws/frame-transport'
    - Protocol detection: window.location.protocol === 'https:' ? 'wss:' : 'ws:'
    - Match Lit implementation at front-end/src/lit/infrastructure/websocket/WebSocketFrameTransportService.ts

    **Error Handling:**
    - Handle WebSocket errors, call errorCallback with descriptive error message
    - Handle connection failures, set state to 'failed'
